强制委派 `vision-analyzer` 子代理分析指定图片，返回纯文本分析结果。

**重要**: 此命令**必须**通过子代理执行视觉分析，禁止父代理直接处理图片数据。这是为了避免图片数据污染父代理的 session 上下文。

## 输入参数

`/analyze-image <图片路径> [分析问题]`

- **图片路径** (必需): 图片文件的绝对路径或相对路径
- **分析问题** (可选): 你想让子代理回答的问题。如果不提供，默认进行详细描述。

## 示例

```bash
# 基本用法 - 详细描述图片
/analyze-image /tmp/screenshot.png

# 带问题
/analyze-image /tmp/login-page.png "页面上有哪些表单字段？"

# 对比两张图片
/analyze-image /tmp/before.png "对比 /tmp/after.png，有什么变化？"

# 使用相对路径
/analyze-image ./screenshots/dashboard.png "检查数据是否正确显示"
```

## 执行步骤

1. **解析输入参数**
   - 提取图片路径（第一个参数）
   - 提取分析问题（剩余参数，如果没有则使用默认问题："请详细描述这张图片的内容，包括布局、元素、文本和颜色。"）

2. **验证图片路径**
   - 检查文件是否存在
   - 如果路径是相对路径，转换为绝对路径
   - 如果文件不存在，提示用户检查路径

3. **委派子代理执行分析**
   
   **必须使用子代理**:
   ```
   使用 Agent 工具委派 vision-analyzer 子代理，传入以下参数：
   
   - agent_type: "vision-analyzer"
   - prompt: "分析图片 <图片绝对路径>，回答：<分析问题>"
   - tools: ["Read", "Bash"]
   ```

4. **等待子代理返回结果**
   - 子代理会使用 `Read` 工具读取图片（模型自动识别）
   - 如果 Read 失败，子代理会使用 Python 脚本兜底：
     ```bash
     python /path/to/vision_analyze.py "<图片路径>" "<分析问题>"
     ```
   - 如果 Python 脚本也失败，使用 hermes 兜底：
     ```bash
     hermes chat -q "分析 <图片路径>，<分析问题>"
     ```

5. **返回分析结果**
   - 子代理返回纯文本分析结果
   - 将结果直接返回给用户
   - **不要**在父代理中处理任何图片数据

## 输出格式

```
📸 **图片分析结果**

**图片路径**: <图片路径>
**分析问题**: <分析问题>

---

<子代理返回的分析结果>

---

✅ 分析完成（由 vision-analyzer 子代理执行）
```

## 防护规则

⚠️ **严格遵守**:

1. **禁止父代理直接读取图片**
   - 不要使用 `Read` 工具读取图片文件
   - 不要在父代理的上下文中包含任何图片数据或 base64 编码

2. **必须通过子代理**
   - 使用 `Agent` 工具委派 `vision-analyzer`
   - 确保 `agent_type` 设置为 `"vision-analyzer"`

3. **只传递路径和问题**
   - 传递给子代理的只有图片路径和分析问题
   - 不要传递图片内容

4. **返回纯文本**
   - 子代理返回的必须是纯文本
   - 如果子代理返回了非文本数据，丢弃并重新请求

## 错误处理

- **文件不存在**: 提示 "❌ 图片文件不存在：<路径>，请检查路径是否正确。"
- **子代理失败**: 提示 "❌ vision-analyzer 子代理执行失败，尝试使用 Python 脚本兜底..."
- **所有方法失败**: 提示 "❌ 所有视觉分析方法都失败了。请检查图片格式是否支持（PNG/JPG/GIF），或手动使用其他工具查看图片。"

## 技术细节

**子代理配置** (vision-analyzer.md):
- **模型**: sonnet（支持视觉）
- **工具**: Bash, Read
- **隔离级别**: 完全隔离（独立 session）

**为什么必须用子代理**:
- 当前主代理（如 qwen3.7-max）不支持视觉
- 直接在主代理中处理图片会导致 API 400 错误
- 图片数据会污染 session 上下文，影响后续节点
- 子代理使用独立 session，处理完后只返回文本结果
