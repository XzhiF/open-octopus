# 需求简报生成器

## 触发条件
Stage 3 agent 节点，在需求澄清和验证方式澄清都完成后加载。

## 职责
读取澄清产物，输出一份 **言简意赅** 的结构化简报。下游所有节点（spec-design、plan-spec、execution、ship）只读此文件，不再回溯原始澄清文件。

## 输入

- `.octopus/xzf/{feature}/00-init/idea.md` — 原始 Idea
- `.octopus/xzf/{feature}/02-clarification/questions.md` — 需求澄清结果
- `.octopus/xzf/{feature}/02-clarification/verification.md` — 验证方式澄清结果
- `.octopus/xzf/{feature}/00-init/workspace-topology.md` — Workspace 拓扑

## 输出

`.octopus/xzf/{feature}/02-clarification/brief.md`

## 生成原则

1. **只保留结论** — 不保留讨论过程、备选方案、已否决选项
2. **每条信息只出现一次** — 不在多个章节重复
3. **可验证** — 每个验收标准必须有具体验证方法
4. **可执行** — agent 读完 brief 就能开始工作，不需要回看原始文件

## brief.md 模板

```markdown
# 需求简报

> Feature: {feature slug}
> 生成时间: {ISO timestamp}
> 分支: {branch}

## 需求概述
[一句话描述核心需求]

## 涉及项目
- [ ] {project-1}（{角色：后端/前端/全栈}）
- [ ] {project-2}（{角色}）

## 功能边界
**做什么：**
- [功能点 1]
- [功能点 2]

**不做什么：**
- [排除项 1]

## 关键决策
| # | 决策 | 结论 | 原因 |
|---|------|------|------|
| D1 | [决策问题] | [选定方案] | [一句话原因] |

## 数据模型变更（如有）
| 表 | 操作 | 字段/说明 |
|----|------|----------|

## API 契约（如有）
| 方法 | 路径 | 端 | 入参 | 出参 | 说明 |
|------|------|----|------|------|------|

## 设计稿（如有）
- Figma: [URL 或 "无"]
- 还原精度: [像素级 / 大致对齐]

## 验收标准
| # | 用户故事 | 验收条件 | 验证方法 |
|---|---------|---------|---------|
| AC1 | As a [角色], I want [功能], so that [收益] | [具体可验证条件] | [验证方式] |

## 验证策略

### 全局配置
- 验证环境：[UAT / 本地 / ...]
- 测试用户：[账号信息]
- 测试数据前缀：[如 "E2E_TEST_"]

### 每层验证方法

#### 单测
- 运行命令：`[具体命令]`
- Mock 对象：[列表]

#### 集成测试
- 接口：[API 路径]
- DB 验证：`SELECT * FROM [表] WHERE [条件]`
- 交叉校验：API 响应 ↔ DB 数据

#### 浏览器 E2E（如有）
- 页面：[URL]
- 操作：[步骤]
- 断言：[预期结果]

#### 手动验证清单（如有）
- [ ] [验证项]

### 敏感凭据
| 用途 | 变量名 | 值 |
|------|--------|-----|
| [如 DB 连接] | [ENV_VAR] | [值或 "待用户提供"] |

### 前置条件
- [ ] [条件 1]

## 风险与注意事项
- R1: [风险点]

## 术语表（如有）
| 术语 | 含义 |
|------|------|
```

## 章节裁剪规则

- **无数据模型变更** → 删除「数据模型变更」章节
- **无 API** → 删除「API 契约」章节
- **无设计稿** → 删除「设计稿」章节
- **无 E2E** → 删除「浏览器 E2E」章节
- **无手动验证** → 删除「手动验证清单」章节
- **无新术语** → 删除「术语表」章节
- **无敏感凭据** → 保留章节但标注「无需凭据」

## 质量检查

生成后自查:
1. 每个验收标准是否有对应验证方法？
2. 关键决策是否都有结论？
3. 功能边界是否明确了「不做什么」？
4. 凭据信息是否完整（如有）？
5. brief.md 是否自包含 — agent 不需要回看原始文件？
