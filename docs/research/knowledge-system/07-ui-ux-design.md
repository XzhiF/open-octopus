## 3. AI 助手面板

```
┌──────────────────────────┬───────────────────────────────┐
│  审核队列                 │  🤖 AI 助手                    │
│                           │                               │
│  ☑ "Promise.allSettled   │  我来帮你审核这条规则。        │
│     聚合调用必须设        │                               │
│     全局超时"             │  当前规则:                    │
│     来源: blocker B4     │  "Promise.allSettled 聚合调用  │
│                           │   必须设全局超时"              │
│  ☑ "stale 检测一律用     │                               │
│     updated_at"          │  💬 你可以说:                  │
│     ⚠️ 与已有规则重叠    │  - "改成只对 health check 端点 │
│                           │    生效"                       │
│  ☐ "scan 节点 Grep      │  - "太严格了，改成建议"        │
│     优先再 Read"         │  - "加上超时的推荐值"          │
│                           │  - "这条 OK"                  │
│                           │                               │
│                           │  ── 对话 ──                   │
│                           │                               │
│                           │  👤: 加上推荐的超时值          │
│                           │                               │
│                           │  🤖: 已修改为:                │
│                           │  "Promise.allSettled 聚合调用 │
│                           │   必须设全局超时，推荐         │
```

- 当前正在审核的规则

- 本次执行的 review blockers / E2E 结果
- 用户偏好

- AI 助手可折叠

## 6. 通知与提醒

| 场景 | 通知方式 | 内容 |
|------|---------|------|
| 手动执行完成 | Dashboard toast | "执行完成，3 条规则待审核" |
| Scheduler 执行积累 | Dashboard badge | 知识 Tab 显示 🔴 N |
| Agent 对话中提议 Skill | 对话内卡片 | Skill 提议卡片 + 操作按钮 |
| 知识文件超 100 行 | Dashboard toast | "octopus.md 超 100 行，建议整理" |
| 重复踩坑自动通过 | Agent 对话通知 | "自动纳入规则: ... (出现 ≥ 3 次)" |
